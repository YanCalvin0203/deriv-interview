NovaPay Security Policy

1. Account Security
Merchants are required to enable two-factor authentication (2FA) within 30 days of account creation. Supported 2FA methods include authenticator apps (TOTP) and SMS. Hardware security keys (FIDO2/WebAuthn) are available for Pro and Enterprise accounts.

Passwords must be at least 12 characters and include a mix of uppercase, lowercase, numbers, and symbols. Passwords are hashed with bcrypt before storage.

2. API Key Management
API keys are generated per environment (sandbox and production). Keys should be stored securely and never committed to version control. Compromised keys must be rotated immediately via the NovaPay dashboard or by contacting support.

API requests must include the Authorization: Bearer <api_key> header. All API traffic is served over HTTPS only; HTTP requests are rejected.

3. Vulnerability Disclosure
NovaPay operates a responsible disclosure program. Security researchers who discover vulnerabilities are encouraged to report them to security@novapay.example.com. NovaPay acknowledges reports within 48 hours and aims to patch critical vulnerabilities within 7 days.

4. Compliance
NovaPay maintains the following certifications:
- PCI DSS Level 1 (Payment Card Industry Data Security Standard)
- ISO 27001 (Information Security Management)
- SOC 2 Type II

Annual third-party penetration tests are conducted, and results are available to Enterprise customers under NDA.

5. Incident Response
In the event of a data breach, affected merchants will be notified within 72 hours as required by GDPR and applicable regulations. NovaPay maintains an incident response team available 24/7.
